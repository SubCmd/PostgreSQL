### Lv.1 — 개념 확인

**1-1. (O/X)** CUBE는 인수의 나열 순서를 바꾸면 결과 행 수가 달라진다.

> 정답 : X, CUBE는 인수의 나열 순서와 무관한 특성을 가지고 있다.
> CUBE(a, b) = GROUPING SETS((a, b), (a), (b), ())로 a,b 하나, a 소계, b 소계, 총계를 제공
> 단, ROLLUP(a,b)와 ROLLUP(b,a)는 다름 각각, GROUPING SETS((a,b), (a), ()), GROUPING SETS((b,a), (b), ())


**1-2. (빈칸)** ROLLUP 결과에서 소계·총계 때문에 생긴 NULL과 실제 데이터의 NULL을 구분하려면 ______ 함수를 사용하며, 초집계 행에서 ______ 을 반환한다.

> 정답 : GROUPING, 1. (실제 데이터 NULL이면 0)

**1-3. (O/X)** `ROLLUP(A, B)`는 `GROUPING SETS((A, B), (A), ())`와 동일한 결과를 반환한다.

> 정답 : O, ROLLUP은 소매를 걷는 것처럼 뒤에 있는 문자에 해당하는 부분을 제거하면서 소계, 총계를 제공

**1-4. (O/X)** 윈도우 함수는 WHERE 절에서 직접 사용할 수 있다.

> 정답 : X
> 윈도우 함수는 SELECT절과 ORDER BY 절에서만 사용 가능
> 인라인뷰로 감싸야함.

### Lv.2-A — 행 수 계산 6문항 ★진단 오답 재검증 구간★
 
모든 문항은 위 [ORDERS] 테이블 기준. **풀이 시 반드시 "공식 → 대입 → 답" 순서로 손으로 계산할 것.**
 
**A-1.** `GROUP BY ROLLUP(CHANNEL, DEVICE)` 의 결과 행 수는?

> 정답 : 총 6행
> CHANNEL | DEVICE | 근거
> --------+--------+--------+-----
> organic | mobile | ┐
> organic | pc     | │ 상세 3행
> paid    | mobile | ┘
> organic | NULL   | ┐ CHANNEL 소계 2행
> paid    | NULL   | ┘
> NULL    | NULL   |   총계 1행

**A-2.** `GROUP BY CUBE(CHANNEL, DEVICE)` 의 결과 행 수는?

> 정답 : (C,D) = 3, (C) = 2, (D) = 2, () = 1
> 총 8개 행
> A-1과 달리 (NULL, mobile), (NULL, pc)가 추가됨.

**A-3.** `GROUP BY GROUPING SETS(CHANNEL, DEVICE)` 의 결과 행 수는?

> 정답 : (C) = 2, (D) = 2
> 총 4개 행
> GROUPING SETS 함수는 나열한 집계 함수만 만든다!!

**A-4.** `GROUP BY ROLLUP((CHANNEL, DEVICE))` 의 결과 행 수는? (괄호 주의)

> 정답 : (C,D) = 3, () = 1
> 총 2개 행
> CHANNEL 소계가 생기지 않음. 

**A-5.** `GROUP BY ROLLUP(CHANNEL, DEVICE, REGION)` 의 결과 행 수는?

> 정답 : (C,D,R) = 4, (C,D) = 3, (C) = 2, () = 1
> 총 10개 행
> 상세 -> 2단 소계 -> 1단 소계 -> 총계

**A-6.** `GROUP BY CHANNEL, ROLLUP(DEVICE)` 의 결과 행 수는?

> 정답 : (C,D) = 3, (C) = 2 
> 총 5개 행
> 고정 컬럼인 CHANNEL은 ROLLUP 진행하지 않음.
> 틀렸음. -> 잘 봐야함.

### Lv.2-B — 윈도우 함수 결과 예측 3문항

**B-1.** 점수 100, 90, 90, 80 (4명)에 대해 `ORDER BY SCORE DESC` 기준으로 RANK / DENSE_RANK / ROW_NUMBER를 각각 적용했을 때, **80점의 값**을 순서대로 쓰시오.

> 정답 : 100, 90, 90, 80
> RANK : 1, 2, 2, 4 / 4
> DENSE_RANK : 1, 2, 2, 3 / 3
> ROW_NUMBER : 1, 2, 3, 4 / 4 (순서대로)


**B-2.** 아래 SQL의 결과는?
 
```
[DAU]
DT   | CNT
-----+-----
d1   | 100
d2   | 120
d3   | 90
```

```sql
SELECT DT, CNT,
       LAG(CNT, 1, 0) OVER (ORDER BY DT) AS PREV_CNT
FROM DAU;
```

> 정답 : 
> DT | CNT | PREV_CNT
> ---+-----+---------
> d1 | 100 | 0        ← 이전 행 없음 → 3번째 인자(기본값) 0
> d2 | 120 | 100
> d3 |  90 | 120


**B-3.** 아래 두 SQL의 결과 차이를 설명하시오. (동일 값 구간 주목)
 
```
[S]
DT | AMT
---+-----
d1 | 100
d2 | 100
d2 | 200
d3 | 300
```
 
```sql
-- (가)
SUM(AMT) OVER (ORDER BY DT ROWS  BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW)
-- (나)
SUM(AMT) OVER (ORDER BY DT RANGE BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW)
```

> 정답 : 
> DT | AMT | (가) ROWS | (나) RANGE
> ---+-----+----------+-----------
> d1 | 100 | 100      | 100
> d2 | 100 | 200      | 400   ← DT가 같은 두 행을
> d2 | 200 | 400      | 400   ← 한 묶음으로 합산
> d3 | 300 | 700      | 700

- ROWS : 물리적 행 단위 누적 (d2 두 행 값이 서로 다름. 단, 동일 DT 내 행 순서는 비결정적!)
- RANGE : ORDER BY 값(DT)이 같은 행은 같은 누적값을 가짐.

### 윈도우 함수 추가 학습

```sql
함수명(컬럼) OVER (
    PARTITION BY 그룹컬럼
    ORDER BY 정렬컬럼
    ROWS/RANGE 윈도우범위
)

-- 윈도우 함수 예시
SELECT 
    emp_name, 
    dept_name, 
    salary,
    AVG(salary) OVER (PARTITION BY dept_name) AS dept_avg_salary
FROM employees;


-- LAST_VALUE 예시
: 가장 마지막 행 값 가져옴

SELECT 
    emp_name, dept_name, hire_date, salary,
    
    -- 잘못된 사용 (현재 행이 마지막이 됨)
    LAST_VALUE(salary) OVER (
        PARTITION BY dept_name ORDER BY hire_date
    ) AS wrong_last_val,

    -- 올바른 사용 (범위를 파티션 전체 끝까지 확장)
    LAST_VALUE(salary) OVER (
        PARTITION BY dept_name ORDER BY hire_date
        ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
    ) AS correct_last_val
FROM employees;


-- ROWS/RANGE

SELECT 
    sale_date, amount,
    
    -- ROWS: 물리적으로 1행씩 더함
    SUM(amount) OVER (
        ORDER BY sale_date 
        ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
    ) AS sum_rows,
    
    -- RANGE: 같은 날짜(값)는 같은 범위로 묶어서 동시 계산
    SUM(amount) OVER (
        ORDER BY sale_date 
        RANGE BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
    ) AS sum_range
FROM sales;


-- LAG
LAG(컬럼, [offset], [default])
offset : 몇 단계 이전 행?
default : 이전 행이 없을 경우 대신 채울 값?

SELECT 
    sale_date,
    amount,
    
    -- 1일 전 매출 가져오기 (이전 행이 없으면 0 처리)
    LAG(amount, 1, 0) OVER (ORDER BY sale_date) AS prev_amount,
    
    -- 당일 매출 - 전일 매출
    amount - LAG(amount, 1, 0) OVER (ORDER BY sale_date) AS diff_amount
FROM daily_sales;
```
