## 0. 전이 가능성 체크 (PostgreSQL 경험자용)
 
| 문법 | Oracle | PostgreSQL | 전이 여부 |
|---|---|---|---|
| 서브쿼리 (단일행/다중행/상관/인라인 뷰/스칼라) | 지원 | 지원 | ✅ 그대로 전이 |
| UNION / UNION ALL / INTERSECT | 지원 | 지원 | ✅ |
| **MINUS** | 지원 | **EXCEPT** 로 존재 | ⚠️ 명칭 암기 (동작은 동일) |

| REGEXP_LIKE 등 정규표현식 함수 | 지원 | `~` 연산자/함수로 존재 | ⚠️ 함수명 암기 |

## 1. 서브쿼리 — 개념 정리
 
### 1-1. 용어 풀이
 
| 용어 | 풀어서 설명 | PM/분석가 비유 |
|---|---|---|
| **단일행 서브쿼리** | 결과가 1행 → `=, >, <` 등 단일행 비교 연산자와 사용 | "평균값 하나 구해서 비교" |
| **다중행 서브쿼리** | 결과가 여러 행 → `IN, ANY(SOME), ALL, EXISTS` 필수 | "VIP 유저 목록에 포함되는지 확인" |
| **상관 서브쿼리** | 바깥 쿼리의 행을 **한 줄씩 참조**하며 반복 실행됨 | 유저별로 "본인 코호트 평균"과 비교하는 루프 |
| **인라인 뷰** | FROM 절 안의 서브쿼리 — 임시 테이블처럼 사용 | 피벗 전 스테이징 쿼리 (dbt의 CTE 모델과 같은 역할) |
| **스칼라 서브쿼리** | SELECT 절 안의 서브쿼리 — 반드시 **1행 1컬럼** | 각 행 옆에 "전체 평균" 컬럼 하나 붙이기 |
 
### 1-2. 다중행 연산자 요약
 
| 연산자 | 의미 (`SAL > ANY (100, 200)` 기준) |
|---|---|
| `> ANY` | 최솟값보다 크면 참 → `SAL > 100` |
| `> ALL` | 최댓값보다 크면 참 → `SAL > 200` |
| `IN` | `= ANY` 와 동일 |
| `EXISTS` | 결과 행이 **존재하기만 하면** 참 (값 비교 안 함, 1건 찾으면 중단) |
