## 1. 인덱스 (Index)

인덱스는 책의 "찾아보기"처럼, 특정 컬럼 값을 빠르게 찾도록 미리 정렬해둔 자료구조다.

```sql
-- 기본 B-tree 인덱스 (=, <, >, BETWEEN, ORDER BY에 효과)
CREATE INDEX idx_orders_user ON orders(user_id);
CREATE INDEX idx_orders_date ON orders(order_date);

-- 복합 인덱스 (여러 컬럼) — 컬럼 순서가 중요!
CREATE INDEX idx_orders_user_date ON orders(user_id, order_date);

-- 부분 인덱스 (특정 조건 행만 인덱싱 → 작고 빠름)
CREATE INDEX idx_orders_completed ON orders(order_date)
WHERE status = 'completed';

-- 인덱스 확인 / 삭제
\d orders                       -- (psql) 인덱스 목록 포함
DROP INDEX IF EXISTS idx_orders_date;
```

### 인덱스 종류

| 종류 | 적합한 경우 |
|------|------------|
| **B-tree** (기본) | 등호·범위·정렬 (`=`, `<`, `BETWEEN`, `ORDER BY`) |
| **GIN** | `JSONB`, 배열, 전문검색(다중 값 포함) |
| **GiST** | 범위·기하·근접 검색 |
| **Hash** | 등호(`=`)만 (거의 B-tree로 충분) |
