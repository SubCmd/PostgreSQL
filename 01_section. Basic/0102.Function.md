# Phase 1-2: 함수와 조건 분기

> **목표**: 조회한 데이터를 **가공**한다. 문자열·숫자·날짜 함수로 값을 변형하고, `CASE WHEN`으로 조건 분기(라벨링), `COALESCE/NULLIF`로 NULL을 안전하게 다룬다.
> **선수 조건**: Phase 1 완료 (SELECT/WHERE/정렬 자유 구사)
> **SQLD 연계**: 2과목 — 단일행 함수, NULL 처리 함수, CASE 표현식


## 1. 문자열 함수

### 1-1. 연결과 변형
```sql
-- || : 문자열 연결 (PostgreSQL 표준)
SELECT name || '(' || channel || ')' AS 회원_채널
FROM users;
-- 결과 예) "김민준 (organic)"

-- CONCAT() : NULL을 빈문자로 처리하며 연결 (|| 는 NULL이 섞이면 전체가 NULL)
SELECT CONCAT(name, ' / ' , country) AS 회원_국가 FROM users;
-- 결과 예) "김민준 / KR"

-- 대소문자 변환
SELECT UPPER(email) AS 대문자, LOWER(name) AS 소문자 FROM users;
-- 결과 예) "EMAIL.COM name"

-- 길이
SELECT name, LENGTH(name) AS 글자수 FROM users;
```

> ⚠️ 'A' || NULL → NULL.
> NULL 가능성이 있는 컬럼을 연결할 땐 CONCAT()이 안전하거나,
> COALESCE로 먼저 채운다(아래 4장).

### 1-2. 자르기, 치환, 추출
```sql
-- SUBSTRING(문자열 FROM 시작 FOR 길이)
SELECT SUBSTRING(email FROM 1 FOR 3) AS 앞3글자 FROM users;

-- LEFT / RIGHT
SELECT LEFT(name, 1) AS 성씨 FROM users;    -- 이름 첫 글자

-- REPLACE(원본, 찾을값, 바꿀값)
SELECT REPLACE(email, '@test.com', '') AS 아이디 FROM users;

-- POSITION : 특정 문자의 위치 (이메일 도메인 추출에 활용)
SELECT
    email,
    SUBSTRING(email FROM POSITION('@' IN email) + 1) AS 도메인
FROM users;
-- 결과: test.com
-- @가 몇번째인지 확인하고 그 다음 내용을 가져옴

-- TRIM : 앞뒤 공백 제거 (데이터 정제 필수)
SELECT TRIM('   hello    ') AS 정제;
```

> 💡 PM 관점
> 이메일 도메인 추출은 "어떤 회사/도메인 사용자가 많은지" 분석에 자주 사용.
> 텍스트 데이터 정제 (`TRIM`, `LOWER`)는 분석 신뢰도의 기본기.


## 2. 숫자 함수

```sql
-- ROUND(값, 소수자리) : 반올림
SELECT
    name, 
    price,
    cost,
    ROUND((price - cost) / price * 100, 1) AS 마진율_퍼센트    
    -- 마진율 = (판매가-원가) / 판매가
FROM products;

-- CEIL(올림) / FLOOR(내림) / ABS(절댓값)
SELECT CEIL(89000.0 / 30000) AS 올림,   -- 3
       FLOOR(89000.0 / 30000) AS 내림,  -- 2
       ABS(-500) AS 절댓값;             -- 500

-- MOD(나머지) / POWER(거듭제곱)
SELECT MOD(10, 3) AS 나머지,      -- 1
       POWER(2, 10) AS 거듭제곱;  -- 1024
```

> ⚠️ 정수 나눗셈 함정
> PostgreSQL에서 `5 / 2 = 2`(정수끼리는 정수 결과)
> 소수가 필요하면 `5.0 / 2` 또는 `5 / 2.0`처럼 한쪽을 실수로 만들거나 형변환(아래 3장)


## 3. 날짜, 시간 함수 (코호트 분석의 핵심)

### 3-1. 현재 시각 & 구성요소 추출
```sql
-- 현재 시각/날짜
SELECT NOW() AS 현재시각, CURRENT_DATE AS 오늘;

-- EXTRACT : 날짜에서 연/월/일/요일 등 추출
SELECT
    name,
    signup_date,
    EXTRACT(YEAR  FROM signup_date) AS 가입연도,
    EXTRACT(MONTH FROM signup_date) AS 가입월,
    EXTRACT(DOW   FROM signup_date) AS 요일번호     -- 0=일요일 ... 6=토요일
FROM users;
```

### 3-2. DATE_TRUNC - 기간 단위로 자르기 (코호트 그룹핑)
```sql
-- 가입일을 '월 단위'로 절삭 → 가입월 코호트의 기준
SELECT
    name,
    signup_date,
    DATE_TRUNC('month', signup_date)::DATE AS 가입월코호트
FROM users;
-- 2024-01-05, 2024-01-08 → 둘 다 2024-01-01 로 묶임
``` 

> 💡 핵심
> `DATE_TRUNC('month', ...)`로 같은 달 가입자를 하나의 코호트로 묶음
> Phase 9의 코호트 리텐션 분석이 바로 여기서 출발한다.
> ('week', 'quarter', 'year'도 가능)

### 3-2.
```sql
-- 날짜 + 기간 (interval)
SELECT signup_date, signup_date + INTERVAL '30 days' AS 가입30일후 FROM users;

-- AGE : 두 날짜 사이 기간
SELECT name, AGE(CURRENT_DATE, signup_date) AS 가입_경과기간 FROM users;

-- 일수 차이 (날짜끼리 빼면 정수 일수)
SELECT name, CURRENT_DATE - signup_date AS 가입_경과일수 FROM users;

-- TO_CHAR : 날짜를 원하는 문자 포맷으로
SELECT TO_CHAR(signup_date, 'YYYY-MM') AS 가입월문자 FROM users;
``` 


## 4. NULL 안전 처리 함수

### 4-1. COALESCE - 첫 번째 non-NULL 값 반환
```sql
-- cost가 NULL이면 0으로 대체 (계산 깨짐 방지)
SELECT
    name,
    COALESCE(cost, 0) AS 원가_보정,
    price - COALESCE(cost, 0) AS 마진_보정
FROM products;

-- 여러 후보 중 첫 유효값 (channel이 없으면 'unknown')
SELECT name, COALESCE(channel, 'unknown') AS 채널 FROM users;
``` 

> 💡 PM 관점
> NULL을 그냥 두면 `SUM`/`AVG` 등에서 행이 누락되거나 계싼이 왜곡됨.
> `COALESCE`로 명시적 기본값을 주는 게 분석 정확도를 지킴

### 4-2. NULLIF - 특정 값을 NULL로 (0으로 나누기 방지)
```sql
-- 분모가 0이면 NULL로 바꿔 '0으로 나누기' 에러 회피
SELECT
    name,
    price,
    ROUND(cost / NULLIF(price, 0) * 100, 1) AS 원가율
FROM products;
-- price가 0인 행은 NULLIF가 NULL로 만들어 → 나눗셈 결과가 NULL (에러 대신 안전)
``` 


## 5. CASE WHEN - 조건 분기 (라벨링의 핵심)

### 5-1. 검색형 CASE (조건식 평가)
```sql
-- 가격대별 등급 라벨링
SELECT
    name,
    price,
    CASE
        WHEN price >= 100000 THEN '프리미엄'
        WHEN price >= 50000  THEN '미들'
        ELSE '엔트리'
    END AS 가격등급
FROM products;
``` 

> ⚠️ WHEN은 위에서부터 순서대로 평가되어 처음 참인 것에서 멈춤
> 그래서 범위 조건은 큰 값/좁은 조건부터 써야 한다. (순서가 틀리면 모두 첫 조건에 걸림)

### 5-2. 단순형 CASE(값 비교)
```sql
-- 주문 상태를 한글 라벨로
SELECT
    order_id,
    status,
    CASE status
        WHEN 'completed' THEN '완료'
        WHEN 'cancelled' THEN '취소'
        WHEN 'refunded'  THEN '환불'
        ELSE '기타'
    END AS 상태_한글
FROM orders;
``` 

### 5-3. CASE + 집계 미리보기 (조건부 카운팅 패턴)
```sql
-- 행을 분류해두면 나중에 GROUP BY로 집계하기 쉬워진다.
SELECT
    name,
    channel,
    CASE WHEN channel = 'paid' THEN '유료획득' ELSE '비유료' AS 획득구분
FROM users;
``` 

> 💡 `CASE`는 SQL에서 가장 활용도 높은 표현식 중 하나
> "조건부 라벨 → GROUP BY 집계", "조건부 합계(`SUM(CASE WHEN ... THEN 1 ELSE 0 END)`)" 패턴은
> Phase 3에서 퍼널·전환율 계산에 핵심으로 쓰인다.

### 6. 형변환(CAST)
```sql
-- CAST(값 AS 타입) 또는 값::타입 (PostgreSQL 축약)
SELECT
    '123'::INTEGER    AS 문자를숫자로,
    price::TEXT       AS 숫자를문자로,
    signup_date::TEXT AS 날짜를문자로
FROM products, users
LIMIT 1;

-- 정수 나눗셈 함정 해결
SELECT 5 / 2            AS 정수나눗셈,  -- 2
       5::NUMERIC / 2   AS 실수나눗셈;  -- 2.5
``` 

### 7. 종합 예시(실무형)
```sql
-- "상품별 마진율 이포트: 등급 라벨 + 마진율 + NULL 안전 처리"
SELECT
    name AS 상품명,
    category AS 카테고리,
    price AS 판매가,
    COALESCE(cost, 0) AS 원가,
    ROUND((price - COALESCE(cost, 0)) / NULLIF(price, 0) * 100, 1) AS 마진율,
    CASE
        WHEN (price - COALESCE(cost, 0)) / NULLIF(price, 0) >= 0.6 THEN '고마진'
        WHEN (price - COALESCE(cost, 0)) / NULLIF(price, 0) >= 0.4 THEN '중마진'
        ELSE '저마진'
    END AS 마진등급
FROM products
ORDER BY 마진율 DESC;
```
