## 1. 집계 함수 (전체 요약)

```sql
-- 전체 회원 수, 상품 평균가, 최고/최저가
SELECT
    COUNT(*)        AS 회원수,            -- 행 개수
    COUNT(channel)  AS 채널있는회원수,     -- NULL 제외 개수
    COUNT(DISTINCT channel) AS 채널종류수  -- 중복 제거 개수
FROM users;

SELECT
    COUNT(*)    AS 상품가,
    AVG(price)  AS 평균가,
    MIN(price)  AS 최저가,
    MAX(price)  AS 최고가,
    SUM(price)  AS 가격합계,
FROM products;
``` 

### COUNT의 3가지 형태 (중요)

| 형태 | 의미 |
|------|------|
| `COUNT(*)` | 전체 행 수 (NULL 포함) |
| `COUNT(컬럼)` | 해당 컬럼이 **NULL이 아닌** 행 수 |
| `COUNT(DISTINCT 컬럼)` | 중복 제거한 고유값 수 |
 
> 💡 **PM 관점**
> `COUNT(DISTINCT user_id)`는 "순 사용자 수(unique users)"의 핵심.
> DAU/WAU/MAU가 전부 이 패턴임.
> `COUNT(*)`(이벤트 건수)와 헷갈리면 지표가 완전히 달라진다.
 
> ⚠️ `AVG`, `SUM` 등은 **NULL을 자동으로 무시**한다. NULL을 0으로 취급하고 싶으면 `AVG(COALESCE(cost,0))`처럼 먼저 채워야 결과가 달라진다.


## 2. GROUP BY - 그룹별 집계

```sql
-- 채널별 회원 수
SELECT channel, COUNT(*) AS 회원수
FROM users
GROUP BY channel
ORDER BY 회원수 DESC;

-- 카테고리별 상품 수 / 평균가 / 최고가
SELECT
    category,
    COUNT(*)          AS 상품수,
    ROUNG(AVG(price)) AS 평균가,
    MAX(price)        AS 최고가
FROM products
GROUP BY category
ORDER BY 평균가 DESC;
``` 

### GROUP BY 황금률 ⚠️
> **`SELECT`에 등장하는 비집계(non-aggregate) 컬럼은 반드시 `GROUP BY`에 들어가야 한다.**

```sql
-- ❌ 에러: name은 집계도 아니고 GROUP BY에도 없음
SELECT
    channel,
    name,
    COUNT(*)
FROM users
GROUP BY channel;

-- ✅ 올바름: 집계 컬럼이거나 GROUP BY에 있는 컬럼만 SELECT
SELECT
    channel,
    COUNT(*),
FROM users
GROUP BY channel;
``` 

> 이 규칙은 어기면 PostgreSQL은 명확히 에러를 낸다(MySQL은 조용히 이상한 값을 주기도 함).
> "그룹당 한행"이 결과이므로, 그룹마다 값이 여러 개인 컬럼은 집계하거나 그룹 기준이어야 한다.

### 가공한 표현식으로 그룹핑 (Phase 2 연계)
```sql
-- 가입월별 신규 회원 수 (코호트 사이즈!)
SELECT
    TO_CHAR(signup_date, 'YYYY-MM') AS 가입월,
    COUNT(*) AS 신규회원
FROM users
GROUP BY TO_CHAT(signup_date, 'YYYY-MM')
ORDER BY 가입월;

-- 국가 x 채널 (다중 컬럼 그룹핑)
SELECT country, channel, COUNT(*) AS 회원수
FROM users
GROUP BY country, channel
ORDER BY country, channel;
``` 


## 3. HAVING - 그룹에 조건 걸기
`WHERE`는 그룹화 전 개별 행을, `HAVING`은 그룹화 후 집계 결과를 필터함.

```sql
-- 회원이 2명 이상인 채널만
SELECT channel, COUNT(*) AS 회원수
FROM users
GROUP BY channel
HAVING COUNT(*) >= 2;

-- WHERE + HAVING 함께: 한국 회원만 집계 후, 평균가... (예시는 products)
-- 평균가가 5만원 넘는 카테고리만 (단, 전자/생활 카테고리 행만 대상)
SELECT category, ROUND(AVG(price)) AS 평균가
FROM products
WHERE category IN ('electronics', 'lifestyle')
GROUP BY category
HAVING AVG(price) > 50000;
```

### WHERE vs HAVING 핵심 차이

| | WHERE | HAVING |
|--|-------|--------|
| 시점 | 그룹화 **전** | 그룹화 **후** |
| 대상 | 개별 행 | 집계 결과(그룹) |
| 집계함수 | 사용 불가 | 사용 가능 |

> 💡 실행 순서
> `FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY`
> "개별 행으로 거를 수 있으면 WHERE(빠름), 집계 결과로 걸러야 하면 HAVING"이 원칙.


## 4. 조건부 집계 - `SUM(CASE WHEN)` 패턴 ⭐

Phase 2의 `CASE`와 집계가 결합하는 지점. 한 쿼리에서 여러 조건의 카운트/합계를 가로로 만든다.

```sql
-- 채널별 회원 수를 '한 행'으로 가로 펼치기 (피벗 효과)
SELECT
    COUNT(*) AS 전체회원,
    SUM(CASE WHEN channel = 'paid'      THEN 1 ELSE 0 END) AS 유료획득,
    SUM(CASE WHEN channel = 'organic'   THEN 1 ELSE 0 END) AS 오가닉,
    SUM(CASE WHEN channel = 'referral'  THEN 1 ELSE 0 END) AS 추천
FROM users;

-- 주문 상태별 건수 + 완료율
SELECT
    COUNT(*) AS 전체주문,
    SUM(CASE WHEN status = 'completed' THEN 1 ELSE 0 END) AS 완료,
    ROUND(
        SUM(CASE WHEN status = 'completed' THEN 1 ELSE 0 END)::NUMERIC
        / COUNT(*) * 100 / 1
    ) AS 완료율
FROM orders;
``` 

> ⚠️ 비율 계산 시
> `::NUMERIC` 형변환 필수.
> 안하면 정수 나눗셈으로 0이 나옴 (Phase2 함정)

### 퍼널 전환율 (events 단일 테이블)

```sql
-- 단계별 순 사용자 수 → 전환율 (Growth 분석의 정수)
SELECT
    COUNT(DISTINCT CASE WHEN event_name = 'page_view' TEHN user_id END) AS 조회_UV,
    COUNT(DISTINCT CASE WHEN event_name = 'add_to_cart' TEHN user_id END) AS 장바구니_UV,
    COUNT(DISTINCT CASE WHEN event_name = 'checkout' TEHN user_id END) AS 결제시작_UV,
    COUNT(DISTINCT CASE WHEN event_name = 'purchase' TEHN user_id END) AS 구매_UV,
FROM events;
```

> 💡 핵심 패턴
> `COUNT(DISTINCT CASE WHEN 단계 THEN user_id END)`로 각 퍼널 단계의 **순 사용자**를 셈
> `CASE`가 조건 불일치 시 NULL을 반환하고, `COUNT(DISTINCT)`가 NULL을 무시하는 성질을 이용. Phase 9 퍼널 분석의 뼈대다.
