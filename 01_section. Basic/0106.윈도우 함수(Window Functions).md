# Phase 1-6: 윈도우 함수 (Window Functions) ⭐

> **목표**: 행을 합치지 않고 **유지한 채**, 그룹 내 순위·누적·이동평균·직전값을 계산한다. "회원별 첫 주문", "일별 누적 매출", "전월 대비 성장률", "세션 기반 퍼널" — Growth 분석의 가장 강력한 도구.
> **선수 조건**: Phase 1~5 완료 (JOIN·`GROUP BY`·CTE 자유 구사)
> **SQLD 연계**: 2과목 — 윈도우 함수(`RANK`/`ROW_NUMBER`/`LAG`/`LEAD`/누적집계)

## 1. GROUP BY vs 윈도우 함수 (핵심 직관)

```
GROUP BY: 그룹을 '한 줄로 압축'         윈도우 함수: 행을 '유지'하며 계산 추가
┌──────┬─────┐                          ┌──────┬─────┬──────────┐
│channel│합계 │                         │ name │price│ 채널평균  │
├──────┼─────┤                          ├──────┼─────┼──────────┤
│paid  │ 300 │   ← 3행이 1행으로         │김민준 │ 100 │   100    │  ← 행 그대로
│organic│200 │                          │이서연 │ 100 │   100    │
└──────┴─────┘                          │박지후 │ 100 │   100    │
                                        └───────┴─────┴──────────┘
```

> 💡 **핵심**
> `GROUP BY`는 "그룹당 한 행"으로 줄임.
> 윈도우 함수는 **원본 행을 그대로 두고** 옆에 집계/순위 컬럼을 덧붙인다.
> "각 상품과 그 카테고리 평균을 한 줄에 같이 보고 싶다"면 윈도우 함수.

## 2. 기본구조: OVER / PARTITION BY / ORDER BY
```sql
함수() OVER (
    PARTITION BY 그룹기준       -- 어떻게 묶을지 (생략 시 전체가 한 그룹)
    ORDER BY 정렬기준           -- 그룹 내 순서 (순위·누적·LAG에 필수)
)
```

```sql
-- 각 상품의 가격 + 같은 카테고리의 평균가를 한 줄에
SELECT
    name,
    category,
    price,
    ROUND(AVG(price) OVER (PARTITION BY category)) AS 카테고리평균,
    price - AVG(price) OVER (PARTITION BY category) AS 평균과의차이
FROM products
ORDER BY category, price DESC;
```

> `PARTITION BY category`= 카테고리별로 창(window)을 나눠 각 창 안에서 평균을 계산.
> `GROUP BY`와 달리 행이 사라지지 않는다.

`GROUP BY` : 지정한 칼럼을 기준으로 데이터를 완전히 뭉쳐버리기 때문에 행의 개수가 줄어들고 개별 행의 상세 정보가 사라짐.
`PARTITION BY` : 논리적인 구역(창)만 나누어 계산할 뿐, 행을 합치지 않으므로 개별 행의 상세 정보를 유지하면서 집계 값을 추가할 수 있음.

## 3. 순위 함수 - ROW_NUMBER / RANK / DENSE_RANK / NTILE

```sql
-- 카테고리별 가격 순위 (세 함수 차이 비교)
SELECT
    name, category, price,
    ROW_NUMBER() OVER (PARTITION BY category ORDER BY price DESC) AS 행번호,
    RANK()       OVER (PARTITION BY category ORDER BY price DESC) AS 순위,
    DENSE_RANK() OVER (PARTITION BY category ORDER BY price DESC) AS 조밀순위
FROM products;
```

| 함수 | 동점 처리 | 예: 100,100, 90 |
| ---- | -------- | --------------- |
| `ROW_NUMBER` | 무조건 고유 번호 | 1, 2, 3 |
| `RANK` | 동점은 같은 순위, 다음은 건너뜀 | 1, 1, 3 |
| `DENSE_RANK` | 동점 같은 순위, 다음은 안 건너뜀 | 1, 1, 2 |

```sql
-- NTILE: N개 그룹으로 균등 분할 (RFM 점수, 사분위에 활용)
SELECT
    name, price,
    NTILE(4) OVER (ORDER BY price DESC) AS 가격_사분위  -- 1=상위25%
FROM products;
```
